HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • The box model refers to the box that wraps around every HTML element. It consist of: margins, borders, padding, and the actual content.
  • A content indicates the space where the data of the HTML element appears.
  • A padding indicates how much space we want around the content inside an element. The padding is transparent.
  • A border is the space between the margin and padding and that goes around the padding and content.
  • A margin indicates how much space we want around the outside of an element. The marging is transparent.
  • The box of an element can be broadly of two types, block boxes and inline boxes.
  • A box type refers to how the box behaves in terms of page flow and in relation to other boxes on the page.
  • Boxes have an inner display type and an outer display type.
  • A block element always starts on a new line and takes up the full width available.
  • Some HTML elements, such as, div, h1-h6, p, form, header, footer, section use block as their outer display type by default.
  • An inline element does not start on a new line and only takes up as much width as necessary.
  • Some HTML elements, such as, span, a, img, em, strong use inline as their outer display type by default.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: commit all files in the working branch.
  • git commit -m "text-message-about-this-commit": commit all the staged changes to the remote repository.
  • The commit message should be short descriptive messages of the changes implemented. See Git Commit Guidelines.

JavaScript

  • JavaScript is a scripting, multi-paradigm, dynamic programming language with types and operators, standard built-in objects, and methods.
  • It supports object-oriented programming with object prototypes and classes.
  • It also supports functional programming since functions are first-class that can be created via expressions and passed as an argument and returned by other functions.
  • There are 7 primitive data types, such as, Number, BigInt, String, Boolean, Symbol, Undefined and Null. Everything else is known as an Object, such as, Function, Array, Date, RegExp and Error.
  • Everything in JavaScript is an object and can be stored in a variable.
  • JavaScript is case sensitive, so always check for variables, objects and classes name rules and casing. If you are unsure, you can check a variable name.
  • Variables are containers that store values that you start declaring with the let keyword followed by the name, for instance, let myVariable;
  • There is no a definitive concensus whether adding semicolons or not at the end of each statement, but there is a basic guide to semicolons in JavaScript.
  • Control flow is the order in which a computer executes code in a script.
  • The control flow in JavaScript runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • A value is considered falsy if either values 0, -0, On, "", null, undefined, NaN or Boolean with false. All other values, including Boolean with true are considered truthy.
  • A variable is a named container that allows us to store data in our code.
  • An array is a single variable that is used to hold a group of data.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
